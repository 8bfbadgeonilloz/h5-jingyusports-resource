# h5-jingyusports-resource

本仓库用于归档和发布多个独立的 HTML 页面及相关资源文件。这些页面不针对任何特定域名或网站，仅作为通用型前端页面存档使用。

## 项目简介

本项目旨在提供一个结构化的 HTML 页面仓库，方便开发者、设计人员或内容管理者快速访问、测试或引用各类独立 H5 页面。页面内容涵盖常见业务场景，如活动页、信息展示页、工具页等，所有页面均以静态 HTML 形式存放，无需后端服务即可运行。

## 目录说明

```
h5-jingyusports-resource/
├── pages/          # 存放所有独立 HTML 页面
│   ├── page-001/   # 单个页面及其依赖资源
│   ├── page-002/
│   └── ...
├── assets/         # 公共资源文件（样式、脚本、图片等）
├── docs/           # 相关文档或说明
└── README.md       # 本文件
```

- `pages/`：每个子文件夹对应一个独立页面，包含该页面的 HTML 文件及专属资源（如 CSS、JS、图片等）。
- `assets/`：存放多个页面可能共用的资源，如基础样式库、通用工具脚本或共享图片。
- `docs/`：可选，用于存放额外的使用说明、设计稿或变更日志。

## 页面归档说明

- 所有页面均为独立 HTML 文件，可直接在浏览器中打开运行。
- 页面之间无依赖关系，可单独使用或组合引用。
- 资源路径使用相对路径，便于本地或静态服务器部署。
- 页面命名遵循 `page-xxx` 格式，`xxx` 为数字序号，便于识别和索引。

## 使用方式

1. 克隆或下载本仓库到本地。
2. 进入 `pages/` 目录，选择需要的页面文件夹。
3. 直接打开其中的 `.html` 文件即可预览页面效果。
4. 如需部署，将整个仓库上传至静态服务器，或使用任何支持静态文件的托管服务。

## 维护说明

- 新增页面：在 `pages/` 下创建新文件夹，放入 HTML 及资源文件，并确保相对路径正确。
- 更新页面：直接修改对应文件夹内的文件，注意保持资源引用一致性。
- 公共资源：如需新增共享资源，请放入 `assets/` 目录，并在页面中引用。
- 建议在提交前进行基本功能测试，确保页面可正常渲染。
